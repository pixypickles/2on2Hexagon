# AGENTS.md

このリポジトリで作業するAIコーディングエージェント向けの指示です。

## 1. 最優先事項

完成を最優先してください。仕様にない機能を勝手に追加しないでください。

実装前に必ず以下を確認してください。

1. README.md
2. SPEC.md
3. TASK.md
4. このAGENTS.md

## 2. 仕様の優先順位

内容が競合した場合は、以下の順で優先します。

1. ユーザーの最新指示
2. SPEC.md
3. TASK.md
4. README.md
5. 実装上の推測

不明点があっても、ゲーム全体を止めない範囲では最小限の仮実装を行い、コード内にTODOを残してください。

## 3. 禁止事項

初期版では以下を実装しないでください。

- オンライン対戦
- セーブデータ
- アンロック
- 育成
- ガチャ
- 課金
- キャラクター能力差
- チーム能力差
- スタミナ
- ファウル
- オフサイド
- スローイン
- コーナーキック
- PK
- ストーリーモード
- チームスローガン

## 4. アーキテクチャ

特定エンジンに依存しすぎない構造を推奨します。

最低限、以下を分離してください。

- GameState：試合状態、時間、得点
- TeamData：チーム名と見た目
- PlayerController：人間操作
- CPUController：CPU判断
- BallController：移動、反射、回転、高さ
- GoalController：得点判定
- MatchController：試合開始、停止、再開、終了
- SpecialShotController：必殺シュート
- InputController：方向入力とABCD
- UIController：画面表示
- GameConfig：調整可能定数

## 5. 調整値

速度、受付時間、ゲージ量などはコードへ散在させず、GameConfigまたは設定ファイルへ集約してください。

マジックナンバーを避けてください。

## 6. 実装順序

TASK.mdのPhase順に進めてください。

各Phase完了時に以下を行ってください。

- ビルド確認
- 最低限の動作確認
- 既存機能の回帰確認
- TASK.mdのチェック更新
- 変更内容を簡潔に記録

## 7. コード品質

- 関数は短く保つ
- 変数名は役割が分かる名前にする
- 例外やnullを無視しない
- 入力処理とゲームロジックを分離する
- フレームレート依存を避ける
- 物理更新は固定時間刻みを優先する
- コメントは「何をしているか」より「なぜ必要か」を書く

## 8. テスト

可能な範囲で自動テストを作成してください。

優先テスト：

- センターライン越境防止
- 壁反射
- ゴール判定
- 得点後の再配置
- パスと同時の操作交代
- チャージ量と速度
- ブレ球の発生条件
- 通常キャッチとジャストキャッチの違い
- 必殺ゲージ消費
- ステルス中も当たり判定が残ること
- ブレイクショットの吹き飛ばし

## 9. 仮素材

本素材がない場合は、円、長方形、単色、簡易パターンで代用してください。

仮素材を理由にゲームロジックの実装を止めないでください。

チームユニフォームは以下の簡易表現で構いません。

- BLIZZARD FOX：白黒縦縞＋水色ライン
- SALVIDA A：エンジ単色＋白パンツ
- SALVIDA B：ターコイズ単色＋白パンツ
- TAKE-ZO：ピンクとネイビーの太ボーダー＋白パンツ
- 漫チェスターP：濃いネイビー単色

## 10. CPU

CPUは最初から高度にしないでください。

優先順位：

1. 自陣制限を守る
2. ゴールを守る
3. ボールへ反応する
4. シュートする
5. パスする
6. 壁反射を狙う
7. 必殺を使う

EASY、NORMAL、HARDの差は主に反応時間、予測精度、行動選択の賢さで表現してください。露骨な能力補正は避けてください。

## 11. ジャストキャッチ

ジャストキャッチはゲームの中心要素です。

- Cのパンチングジャンプ中でも成立
- Dのキャッチジャンプ中でも成立
- 立ち状態でも成立
- 接触直前のCまたはD入力で成立
- ブレ球と必殺シュートはジャストキャッチでのみ完全保持
- 成功時は約0.2秒のスローまたは停止演出

この仕様を簡略化しないでください。

## 12. 必殺シュート

4種類を別クラスまたは共通インターフェースで実装し、追加や調整が容易な構造にしてください。

- Hook Shot
- Side Net Curve
- Break Shot
- Stealth Shot

必殺シュートもジャストキャッチ可能です。

## 13. UI

タッチボタンは誤入力を防ぐため十分な大きさにしてください。

- 方向入力：画面左
- A、B、C、D：画面右
- 半透明
- 試合画面を覆いすぎない

## 14. 変更報告

作業完了時は以下を報告してください。

- 実装した内容
- 変更した主なファイル
- 動作確認内容
- 残っている問題
- 次に行うべきTASK.mdの項目

## 15. 完成より先に行わないこと

以下は、基本ゲームが最後まで遊べるようになる前に着手しないでください。

- 豪華な演出
- 複雑なメニューアニメーション
- 高度なCPU
- 大量の効果音
- 細かなグラフィック調整
- リファクタリングだけを目的とした大改修

まず遊べるものを作り、その後に磨いてください。
